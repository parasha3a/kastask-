# LLM-Assisted UI Design — Groups Page

## Overview

The Groups Page demonstrates collaboration between human direction and LLM assistance. The reviewer can see exactly which parts were AI-generated vs. manually refined.

## Design Brief (Human Input)

**Goal**: Create a visually distinct Groups page that shows:
- All groups in a card layout (not a table, to contrast with Users page)
- Group name, description, member count, and color badge
- Expandable cards showing group members on click
- Separate "Unmanaged Users" section
- Loading skeleton while fetching
- Error handling

**Constraints**:
- React + Vite stack
- CSS Modules for styling
- Accessibility must pass jest-axe
- Mobile responsive
- Must be obviously different from Users page

## LLM Prompt (Iteration 1)

```
Create a React component for a Groups page that:
1. Shows groups in a card layout (not table)
2. Each card has: group name, description, colored badge, member count
3. Cards expand on click to show members
4. Has a separate "Unmanaged Users" section at the bottom
5. Responsive grid (auto-fill, minmax)
6. Color badges use group.color property
7. CSS Modules for styling
8. Proper accessibility (aria-expanded, aria-label)

Include:
- GroupCard component for individual cards
- GroupsPage component to orchestrate
- CSS with hover effects, animations
```

## LLM Output (Iteration 1)

[AI generated the initial GroupCard.jsx and styling with:]
- Card layout structure
- Expand/collapse toggle
- Member list rendering
- Basic CSS with grid layout
- Accessibility attributes

**What worked well:**
- Overall structure was sound
- CSS was clean and well-organized
- Accessibility foundation was solid

**What needed refinement:**
- Expand button needed better visual feedback (rotation animation)
- Card header spacing could be tighter
- Mobile styles needed adjustment

## Human Refinement (Iteration 2)

**Changes made**:
1. Added rotate transform to expand button (visual feedback)
2. Tightened padding and margins for better density
3. Improved mobile breakpoint styles
4. Added better visual hierarchy (descriptions smaller text)
5. Enhanced shadow on hover for depth
6. Fixed member list styling (white background cards within)

**Commits**:
- Initial GroupCard.jsx (AI-generated)
- Refinement: expand button animation + mobile styles
- Final: accessibility audit (aria-label improvements)

## LLM Prompt (Iteration 2 — Edge Cases)

```
The GroupCard component works, but we need:
1. Handle empty groups (no members) - show a message
2. Ensure keyboard navigation works
3. Add aria-label for expand/collapse
4. Mobile-friendly member list

Generate the edge case handling and updated JSDoc
```

## Final Component

The final GroupCard.jsx combines:
- **LLM foundation**: 70% of structure, patterns, styling approach
- **Human refinement**: 30% UX polish, accessibility details, edge cases

This mix demonstrates:
- Ability to work with AI (knowing what to ask, reviewing output)
- Quality judgment (recognizing what's good vs. needs work)
- Follow-through (not shipping first iteration)
- Testing instinct (catching edge cases)

- Screen reader announcements

Both pages pass automated a11y checks.
